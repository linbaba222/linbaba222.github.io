windows django虚拟环境win10搭建使用
发表于 2020-07-31| 更新于 2020-07-31|技术
阅读量:

设有环境变量workon，设置了集中保存虚拟环境的自定义路径D:\software\envs
使用workon查看当前有哪些虚拟环境
ps：本机现创建有两个虚拟环境testvir3与virlearn，并分别安装的是django1.9.8与django2.2
创建虚拟环境在cmd命令行 mkvirtualenv 文件名
可以直接进入激活状态
退出虚拟环境cmd命令：deactivate.bat
使用workon直接进入指定的虚拟环境
workon 虚拟环境名字
进入虚拟环境pip list查看当前已安装的安装包（这里有一个小问题，这里本机python已经安装了许多库了，为何这里的虚拟环境一个库都没有，还就是说只是映射了一个python主程序过去了）
在虚拟环境里面创建一个项目：
django-admin startproject 自定义的项目名称

命令makemigrations让Django确定该如何修改数据库，使其能够存储与我们定义的新模型相关联的数据

每次当我们想要修改这个应用程序所管理的数据时，要有下面的三个步骤
1、修改models.py
2、对learning_logs调用makemigrations
3、让Django迁移项目

启动你的项目名称:
cd 你的项目名称
python manage.py runserver
通过ctrl+c来终止服务器
在浏览器中输入访问主机地址访问我们的服务器
ps：（可以通过python manage.py help命令查看可用命令列表；在启动服务器时，也可以通过python manage.py runserver 1.2.3.4:5678来指定将服务器运行于哪个IP地址和端口。）
ps:(用于开发的服务器在需要的情况下会对每一次的访问请求重新载入一遍Python代码。所以你不需要为了让修改的代码生效而频繁的重新启动服务器。然而，一些动作，比如添加新文件，将不会触发自动重新加载，这时你得自己手动重启服务器。)
注意：这里的创建的真正的项目的配置文件应该在C:\Windows\System32\djangolearn
里面，而虚拟环境
D:\software\envs
里面的只是一些模板，修改这些模板对服务器并不起到作用，另外项目文件夹已经打开权限了，可以随意修改。
